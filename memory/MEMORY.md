## Lessons Learned

### Error Patterns
1. NEVER do blanket indentation of 700+ lines in a Python file — breaks try/except structure
2. Git checkout reverts to committed version — always check if changes are uncommitted first
3. LLM sometimes returns {"reasoning": "..."} with no structured data — need retry logic
4. Gemini SDK has no default timeout — must pass http_options with timeout=60000
5. Provider timeout chain can waste 5-15 min per call if primary providers are down
6. Canonical JSON extraction (grab any list of dicts) can map wrong data — needs semantic checking
7. MechanismCompleteness checker is physics-only — needs domain-awareness for bio/ecology
8. Literature search uses LLM reframe as query — generic reframes return irrelevant papers

### Successful Strategies
1. Recurrent loop (Phases 6-8, 3 iterations) significantly improves theory quality
2. Math Chain (Phase 6.5) with domain-awareness works for physics domains
3. Multi-provider routing with retry loop ensures pipeline never hangs
4. Claude Fable 5 produces dramatically better curious questioning (12 questions vs 3)
5. Domain-specific keyword extraction for literature search prevents irrelevant papers
6. LLM reasoning retry (use reasoning text as context for second call) recovers failed extractions
7. Score dampening (0.7 * prev + 0.3 * current) prevents score explosion across loops
